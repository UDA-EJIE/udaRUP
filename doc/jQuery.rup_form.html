rup_form

Permite al usuario introducir datos en una serie de campos para ser enviados al servidor y ser procesados.

Source:
  • rup.form.js, line 33
Tutorials:
  • Tutorial: rup.form

Example

var properties={
  // Propiedades de configuración
};

$("#formulario").rup_form(properties);

Members

options

Opciones por defecto de configuración del componente.

Properties:
Name Type Argument Default Description
beforeSerialize jQuery.rup_form~beforeSerialize <optional>
 null

Función de callback que será invocada antes de realizarse la serialización del formulario. Permite la modificación de los datos del formulario antes de que estos sean recuperados para su procesado por el componente.
beforeSubmit jQuery.rup_form~beforeSubmit <optional>
 null

Función de callback que será invocada antes de realizarse el envío del formulario. Permite acceder a la información que será enviada al formulario. En caso de retornar false no se realizará en envío.
clearForm boolean <optional>
 null

Propiedad booleana que determina si el formulario debe de limpiarse después de realizar el envío.
data object <optional>

Mediante esta propiedad es posible especificar parámetros extra que sean enviados alservidor.
dataType string <optional>

Tipo de datos esperados en la respuesta. Los valores posibles son null, xml, json y script.
error jQuery.rup_form~error <optional>

Función de callback que será invocada cuando se produzca un error.
forceSync boolean <optional>
 false

Propiedad booleana. En caso de ser true elimina la corta espera que se produce antes de enviar el formulario cuando se envían ficheros o se utiliza la opción de iframe. La espera se utiliza para permitir al navegador actualizar modificaciones realizadas en el DOM antes de que se realice el
envío de los datos.
iframe boolean <optional>
 false

Determina si el formulario debe de ser enviado siempre mediante un iframe.
iframeSrc string <optional>

Propiedad de texto que deberá ser utlizada siempre en conjunción con la propiedad iframe. Por defecto, about:blank. Por defecto para páginas que utlicen el protocolo https, javascript:false.
iframeTarget string <optional>
 null

Identifica el iframe que será utilizado como target en la respuesta en los envíos de ficheros. Por defecto, el componente creará un iframe temporal para capturar la respuesta del envío de ficheros.
multimodel boolean | object <optional>
 false

Permite especificar la configuración que se deberá de aplicar a la hora de realizar el envío de varias entidades en la misma petición. La configuración de este parámetro se detalla en el apartado 9.2.
replaceTarget boolean <optional>
 false

Opcionalmente se utililiza junto con la opción target. En caso de ser true el elemento identificado en la opción target será reemplazado. En caso de ser false solo su contenido será reemplazado.
resetForm boolean <optional>
 false

Propiedad booleana que determina si el formulario debe ser inicializado al realizarse el envío del mismo.
semantic bolean <optional>
 false

Propiedad booleana que determina si los campos del formulario deben ser enviado en estricto orden semántico. Por defecto la serialización normal del formulario se realiza en orden semántico exceptuando los campos img.
suceess jQuery.rup_form~suceess <optional>

Método callback que será invocado cuando se reciba la respuesta del formulario.
target string | jQuery | Element <optional>

Identifica los elementos de la página que deberán ser actualizados con la respuesta del servidor. El target puede ser indicado mediante un selector de jQuery, un objeto de jQuery o un objeto DOM.
type string <optional>

Detemina el método con el que se enviará el formulario, GET o POST. Por defecto el valor de la propiedad method indicada en el formulario o GET en caso de no encontrarse.
uploadProgress jQuery.rup_form~uploadProgress <optional>
 null

Método que será invocado con información de progreso del envío del formulario (en caso de estar soportado por el navegador).
url string <optional>

URL a la cual se realizará en envío del formulario. Por defecto el valor indicado en la propiedad action del formulario.
useJsonIfPossible boolean <optional>
 true

Mediante este parámetro se especifica al componente que debe de utilizar el formato application/json como prioridad (siempre que sea posible) al realizar el envío del formulario.
